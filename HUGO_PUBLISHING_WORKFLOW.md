# Hugo Publishing Workflow - StillPoint Project

## Architecture Overview

**Production Setup:**
- **CloudFlare Tunnel** (PID 201) → `localhost:8080` → **Hugo Development Server**
- **Git Repository:** `/home/docker/stillpoint-project/`
- **Hugo Site:** `/home/docker/stillpoint-project/site/`
- **Git Hook:** `/home/docker/stillpoint-project/scripts/process-content.sh`

## Publishing Process

### 1. Local Development
```bash
# Make changes to content in site/content/
# Test locally if needed: hugo server -D
```

### 2. Git Commit & Push
```bash
git add .
git commit -m "Your changes"
source ~/.ssh_agent_env  # Ensure SSH keys loaded
git push production master
```

### 3. Automated Production Process
The git post-receive hook automatically:
1. **Processes novel content** from `/novel/scenes/`
2. **Builds Hugo site** (25+ pages)
3. **Kills old Hugo server** processes
4. **Restarts Hugo server** on port 8080
5. **CloudFlare tunnel** serves updated content

## Troubleshooting

### If Site Not Updating After Push
1. **Check Hugo server status:**
   ```bash
   ssh docker@10.10.10.30 "ps aux | grep hugo"
   ```

2. **Manually restart Hugo:**
   ```bash
   ssh docker@10.10.10.30 "cd /home/docker/stillpoint-project/site && pkill -f 'hugo server' && nohup hugo server -p 8080 -D --bind 0.0.0.0 --baseURL http://localhost:8080 > hugo.log 2>&1 &"
   ```

3. **Check git hook logs:**
   ```bash
   ssh docker@10.10.10.30 "tail -50 /home/docker/stillpoint-project/scripts/process-content.sh.log"
   ```

### Common Issues
- **Zombie processes:** Use `killall hugo` to clean up
- **Port conflicts:** Ensure only one Hugo server on port 8080
- **SSH keys:** Load with `ssh-add ~/.ssh/id_rsa_stillpoint`
- **Content not updating:** Verify git push reached production server
